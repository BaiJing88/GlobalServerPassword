# GlobalServerPassword 使用说明

## 一、功能概述

**GlobalServerPassword** 是一款全平台服务器端密码验证模组/插件。玩家在加入服务器后，必须输入正确的全服密码才能进行游戏。未验证玩家将被：

- 施加**失明 + 缓慢**效果
- **完全冻结**在原地（无法移动、跳跃、切换手持）
- 禁止打开任何**容器/背包**、禁止**丢物品/拾取物品**
- 禁止**破坏/放置方块**、禁止**攻击/使用物品/交互**
- 禁止**聊天**和**使用命令**（除登录相关命令外）
- 密码错误达到一定次数或超时会**自动踢出**

## 三、安装方法

### Forge / NeoForge
1. 下载对应版本的 `.jar` 文件
2. 放入服务器的 `mods` 文件夹
3. 启动服务器

### Fabric
1. 下载对应版本的 `.jar` 文件
2. 确保已安装 **Fabric Loader** 和 **Fabric API**
3. 放入服务器的 `mods` 文件夹
4. 启动服务器

### Bukkit / Spigot / Paper
1. 下载 `globalserverpassword-bukkit-1.0.0.jar`
2. 放入服务器的 `plugins` 文件夹
3. 启动服务器

## 四、密码存储说明

密码采用 **SHA-256 哈希值** 存储在配置文件中。**服务器不会保存密码明文**，请务必牢记当前密码。

> **重要提示**：首次启动自动生成的随机密码仅在控制台显示一次。关闭控制台或日志滚动后将无法再次查看。若忘记密码，只能通过 OP 命令重新设置。

## 五、配置文件

首次启动后会自动生成配置文件，关键配置项如下：

```toml
# 密码哈希（请勿手动修改）
password-hash = "..."

# 最后更新时间戳
last-update-time = 0

# 自动更新密码（默认关闭）
auto-update.enabled = false
auto-update.interval-minutes = 60
auto-update.length = 8

# 踢出设置
kick.max-attempts = 3        # 密码错误多少次后踢出
kick.timeout-seconds = 120   # 多少秒未输入密码自动踢出
```

## 六、命令使用

### 玩家命令

#### `/gsp <密码>`
输入全服密码进行验证。验证成功后即可正常游戏。

> 旧版命令 `/gsp login <密码>` 仍然兼容，但推荐使用简化版 `/gsp <密码>`。

### OP / 控制台命令

#### `/gsp setpassword <新密码>`
手动设置新的全服密码。设置后会立即生效，已在线玩家不需要重新输入密码。

#### `/gsp generatepassword`
自动生成随机密码（默认 8 位，包含数字、小写、大写）。

> **注意**：生成的新密码仅会在控制台和 OP 聊天栏显示一次，请务必立即记录。

#### `/gsp generatepassword <长度>`
生成指定长度的随机密码。例如：
```
/gsp generatepassword 12
```

#### `/gsp generatepassword <长度> --numbers true/false --lowercase true/false --uppercase true/false`
高级用法，可控制密码字符集。例如：
```
# 16 位纯字母（无数字）
/gsp generatepassword 16 --numbers false --lowercase true --uppercase true

# 10 位纯数字
/gsp generatepassword 10 --numbers true --lowercase false --uppercase false

# 20 位纯小写
/gsp generatepassword 20 --numbers false --lowercase true --uppercase false
```

> 如果全部关闭，会自动 fallback 为 `a-z` 小写字母。

#### `/gsp clearips`
**清除所有已保存的 IP 白名单**。执行后，所有玩家（包括之前自动登录的玩家）下次加入服务器时都需要重新输入密码。

## 七、游戏内机制详解

### 1. 首次加入服务器
- 若玩家的 IP 已在白名单（上次成功登录的 IP），则**自动通过**，无需输入密码。
- 否则屏幕会陷入失明，同时每 3 秒收到一次金色提示：
  > `请输入全服密码以开始游戏: /gsp <密码>`

### 2. 效果机制
- **首次登录**：立即施加 10 秒失明 + 缓慢
- **持续补效果**：每 5 秒再次施加 10 秒效果，确保不会中断
- **验证成功后**：立即清除失明和缓慢效果，其他药水效果不受影响

### 3. 移动冻结
- 未验证玩家会被**完全钉在原地**
- 无法行走、奔跑、跳跃、甚至被水流/活塞推动
- 无法切换手持物品栏（滚轮/数字键无效）

### 4. 物品与容器限制
- 无法打开箱子、熔炉、工作台等容器
- 无法打开自己的背包（按 E 会立即被关闭）
- 无法丢弃物品（按 Q 无效，物品保留在背包）
- 无法拾取地上的掉落物

### 5. 世界交互限制
- 无法破坏方块
- 无法放置方块
- 无法攻击生物/玩家
- 无法使用物品（右键空气/方块）
- 无法与实体交互（如右键村民）

### 6. 聊天与命令限制
- 无法发送聊天消息
- 无法使用任何命令，**除了**：
  - `/gsp <密码>`
  - `/globalserverpassword ...`
  - `/login` （兼容 AuthMe 等登录插件）
  - `/register` （兼容 AuthMe 等登录插件）

### 7. 自动踢出
- **密码错误过多**：连续输错达到 `kick.max-attempts` 次（默认 3 次）会被踢出，并提示：
  > `密码错误次数过多，已被踢出服务器。`
- **超时未输入**：加入后超过 `kick.timeout-seconds` 秒（默认 120 秒）未输入密码会被踢出，并提示：
  > `未在 120 秒内输入密码，已被踢出服务器。`

## 八、与 AuthMe 的兼容性

本插件/模组与 **AuthMe** 等登录插件完全兼容：

- `/login` 和 `/register` 命令**永远放行**
- 未验证玩家在输入全服密码前已经被冻结和失明，AuthMe 无法抢先执行任何操作
- 如果同时安装了 AuthMe，玩家进服后的流程为：
  1. 输入全服密码 `/gsp <密码>`
  2. 输入 AuthMe 密码 `/login <密码>`

## 九、安全提示

- **密码只显示一次**：无论是首次启动自动生成的密码，还是 `/gsp generatepassword` 生成的密码，都只在生成时显示一次。
- **无法查看当前密码**：服务器不存储明文，因此没有"查看当前密码"的命令。请妥善记录。
- **忘记密码怎么办**：OP 玩家可以在游戏中输入 `/gsp setpassword <新密码>` 来重置。

## 十、常见问题 (FAQ)

**Q: 玩家说输入了密码但一直提示"请先输入全服密码"**
A: 请确保玩家使用的是 `/gsp <密码>`，注意区分大小写。如果仍有问题，请检查服务器日志中是否有命令拦截记录。

**Q: 怎么让所有玩家重新输入密码？**
A: OP 玩家执行 `/gsp clearips`，这会清空所有已记录的 IP 白名单，所有人下次进服都需要重新验证。

**Q: 如何重置密码？**
A: OP 玩家在游戏中输入 `/gsp setpassword <新密码>` 即可。

**Q: 能不能关闭密码错误踢出或超时踢出？**
A: 可以。将配置文件中的 `kick.max-attempts` 改为非常大的数字（如 999），`kick.timeout-seconds` 也改为很大的值（如 99999）即可。

**Q: 自动更新密码是什么意思？**
A: 开启 `auto-update.enabled` 后，服务器会每隔 `interval-minutes` 分钟自动生成一个新密码，并通过控制台和 OP 聊天栏通知（同样只显示一次）。

**Q: 玩家换 IP 后需要重新输入密码吗？**
A: 是的。系统只会记住玩家**最后一次成功登录**的 IP，换 IP 后需要重新验证。
